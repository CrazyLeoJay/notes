## 算法

### 1. 对称加密算法对比

| 算法         | 密钥长度      | 块大小 | 速度  | 安全性   | 应用场景                    | 优缺点                        |
| :----------- | :------------ | :----- | :---- | :------- | :-------------------------- | :---------------------------- |
| **AES**      | 128/192/256位 | 128位  | ⭐⭐⭐⭐  | ⭐⭐⭐⭐⭐    | SSL/TLS, 磁盘加密, 文件加密 | ✅ 安全高效 ⚠️ 需要安全密钥分发 |
| **DES**      | 56位          | 64位   | ⭐⭐⭐   | ❌ 已破解 | 遗留系统                    | ❌ 已不安全 ✅ 兼容性好         |
| **3DES**     | 112/168位     | 64位   | ⭐⭐    | ⭐⭐       | 银行系统, 遗留升级          | ⚠️ 性能差 ❌ 逐渐淘汰           |
| **Blowfish** | 32-448位      | 64位   | ⭐⭐⭐⭐  | ⭐⭐⭐      | 文件加密, 密码管理          | ✅ 快速 ⚠️ 块大小小             |
| **Twofish**  | 128/192/256位 | 128位  | ⭐⭐⭐   | ⭐⭐⭐⭐     | 竞争AES的算法               | ✅ 非常安全 ⚠️ 不如AES普及      |
| **ChaCha20** | 256位         | 流密码 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐    | 移动设备, TLS 1.3           | ✅ 极快 ⚠️ 相对较新             |

### 2. 非对称加密算法对比

| 算法        | 密钥长度    | 等效安全       | 速度  | 安全性 | 应用场景                | 优缺点                      |

### 标准

#### X.509标准

- **目的**：主要用于公钥证书格式
- **组织**：ITU-T（国际电信联盟）制定
- **应用**：数字证书、SSL/TLS、身份验证
- **特点**：定义了公钥的标准化表示方法

#### PKCS#8标准

- **目的**：私钥信息的语法标准
- **组织**：RSA Laboratories
- **应用**：私钥存储和交换
- **特点**：支持密码保护，更安全的私钥存储



| 特性           | X.509公钥        | PKCS#8私钥              |
| :------------- | :--------------- | :---------------------- |
| **设计目的**   | 公开分发和验证   | 安全存储和保密          |
| **安全性**     | 无需加密         | 支持密码加密            |
| **互操作性**   | 极高，行业标准   | 高，广泛支持            |
| **包含信息**   | 算法+公钥数据    | 版本+算法+私钥数据+属性 |
| **文件扩展名** | .crt, .cer, .pem | .key, .p8, .pem         |
