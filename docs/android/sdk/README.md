# Android SDK 构建开发

开发SDK主要是为了对第三方公司提供服务。

这里主要探讨[**手游SDK**](guide)相关的设计与开发，由于各个公司的情况不一样，这里将不涉及代码。指探讨设计方案。

## 题外话

我写这段话的目的是提醒，提醒各位同僚以我的经历为鉴，不要被公司白嫖。就算要做什么提升自身效率的工作，除非是有报酬，也不要上交，在自己电脑上部署，**自己用就好了**，离职时候直接重置电脑，因为你提交了公司不会念你的好，如果因为这些工具别人用的出问题了，还要追你的责任。这些代码本来也不是工作内容，只要保证公司需要交付的文件能按时交付就好。做好本职工作，至于是怎么做的，那是我们自己的手段。

大多数时候，公司管理的事实就是。做的越多错的越多，哪怕你做的成绩再好，他们只会觉得理所当然。一边白嫖你，一边安慰你，又一边暗戳戳骂你傻。

**当然如果真的碰上了那种懂行的老板和领导，奖罚分明，也不要吝啬表现。不过太需要运气了。**

这里写的内容都是我之前的工作经验，关于SDK的**重构**和**框架搭建**，还有**构建脚本**和**自动打包服务（云端长期运行的服务）**的内容都是我真实主导从零到一一点一点做过的，一个也是新到了一个公司，需要展现出能力来获取认可，另一个也是在做这些事情的同时学习了很多方面的知识。但事实上，也是得到了同事们和领导们的认可，并且确实提升了极大的工作效率。可以说所作的已经远远超出一个普通Android岗位所需要干的事情。

但事实上，公司根本不在乎，所有的认可也就是**口头上**，当初为了推进这些事，我曾提过很多次，每次都是怕担责，然后不了了之。当然我也不是无的放矢，原来的SDK也是从别处买来的，几乎每天我们都在处理shi山代码的bug，很多问题都是设计上的缺陷，每次去提也是有理有据的（但事实上，我感觉技术领导听不懂），最后在懂行人的帮助下才推进去做，最后虽然也是有奖励，把我远低于这个岗位的工资水平（非计算机专业，入行工资比较低）拉到和别人一样。我也高兴，以为自己的能力被认可了，我感觉还蛮有盼头的，以我表现出远超当前岗位的水平能力，虽然慢点，但应该也不会太差。当然如果一直是这样，我也不是不能接受，最起码大家都一样，就当是规则腐朽；但新招的人却轻轻松松超过我的工资好多，这个时候就觉得我上当了，上了大当，懂行的朋友就能发现，这些事情本来就不是一个普通Android岗位能做的，我只是感觉到我被公司白嫖了，还在嘲笑我，就像是渣女说的：

“是你非要做，又不是我逼你的，找我干什么。”

这个时候就会有人说了，一看就是纯技术人员不懂人情世故，自己蠢。

这个我不想辩解，我只是觉得如果要做好一件事情，那么我们做的最多的事情就是试错、记录、总结经验。并且还需要正向奖励。如果一味的索取，那么老板就会在短时间内赚的盆满钵满后然后跑路。作为一个受过九年义务教育骗局的人来说，我希望是付出就应该有相应的回报，哪怕只有一点呢？



## 使用到的开发环境和平台

- java 8+
- kotlin
- Gradle
- Gradle plugins
- Android 5.0+ 
- Android SDK Tools (对应版本即可)
- Android Studio
- ApkTools

版本没有太多限制，可以根据实际情况版本调整实现
